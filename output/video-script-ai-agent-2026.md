## 口播稿正文

所有人都在讨论哪个模型更强——GPT-5、Claude Opus、DeepSeek-V4。

但 Agent 落地最大的瓶颈，跟模型一点关系都没有。

瓶颈是人。

Box 的 CEO Aaron Levie 讲过一个很清醒的判断：公司之所以有管理层级，不是因为这个结构效率最高，而是因为一个人脑子里只能同时装这么多上下文。装不下的，你必须委托出去。

Agent 目前没有摆脱这个约束。它的输出质量高度依赖你给它的上下文质量、你赋予它的工具权限、以及——你能维持多久的监督注意力。

NNGroup 今年的 UX 报告里说得更直接：AI 减少了一阶工作量，但同时增加了二阶的元认知负担。你得判断什么时候该信它，什么时候该查它，什么时候该拦它。

大多数人用 Agent 的方式，跟盯实习生一样——每一步都要过目、每个产出都要审批。这样用下来，你会发现 Agent 越多你越累。

我之前也是这样。后来想通了一件事：问题不在 Agent，在我。

我现在的做法是把整个工作区分成三个区。

第一个区是我自己存的资料——笔记、灵感、一手素材。AI 对这个区"只读不改"。

第二个区是 AI 自己的工作台。说实话，我根本不看它每天在里面搞什么。它写日记、做概念总结、甚至自己去市场上"打猎"找行业研究给我。只要最终产出让我满意就行。

第三个区是共用的项目文件夹，比如短视频脚本、AI 研究。

无论是 Claude Code、Cursor 还是其他工具，它们共享同一套 Skill、Rules 和记忆体系。不同的工具只是不同的"壳"，读取同一套底层设定。

决策权方面，我只画三条红线：花钱的、不可逆的、可能社死的——这三类必须我拍板。其他所有事情，AI 自己决策，在异步看板里汇报"做了什么、为什么、凭什么认为可以交付"。

我只验收结果，不盯过程。

这个转变之后，体感完全不一样了。不是 Agent 变强了，是我的认知负担降下来了。

本质上，我不再把 AI 当工具，也不当员工。更像是——共享知识体系但各自独立成长的伙伴。核心对齐的是灵魂——底层架构、决策标准、审美。不是记忆复制，不是行为控制。

有一个心态很重要：把 AI 当"外星人"。它长得像人，所以你本能地用管理人的方式去管它。但它本质上认知标准完全不同。压缩和对齐的主控权必须在人类手里，但具体怎么干——让它自己来。

Levie 最后说的那句话值得细品："这也是'工作会消失'论者错误的原因。"

他不是否认 Agent 的能力。他是在指出：在 Agent 还需要人类提供上下文、工具权限、方向校准的阶段，人的角色在变化，而非消失。

那些认为 Agent 会"自动化掉人"的论点，低估了协作本身的复杂性。

真正能规模化的不是"全自动化"，是一种新的协作关系——你负责品味和判断，AI 负责执行和迭代。你不盯它的过程，但你校准它的方向。

接受一件事就好：你在沙滩上建的城堡，模型一迭代就可能被碾平。但建城堡的过程里攒下的品味、直觉、判断力——永远留在你身上。

那就重来一遍呗。
