# 一稿多投适配 Skill

将已有内容适配到不同平台/格式，保留核心观点，调整结构、语气和篇幅。

原则：适配不是删减，是**重新组织**。不同平台的读者期望不同，同一个观点需要不同的包装。

---

## 适配规则

### 长文 → 短内容
- **字数**：压缩到 300-500 字
- **结构**：提取 1 个核心观点 + 3 个要点（不是缩写全文，是重新选择最有冲击力的部分）
- **标题**：加数字或 emoji，≤ 20 字
- **语气**：从"有深度的聊天"变成"跟朋友分享"
- **金句**：每段确保有一句可截图传播的话
- **结尾**：3-5 个 tag + 封面文案建议
- **删掉**：背景铺垫、过渡句、"首先其次最后"

### 长文 → 短视频口播稿
- **字数**：300-450 字（60-90 秒）
- **结构**：只保留 1 个核心论点 + 1 个最强案例
- **语气**：纯口语，像在跟一个人说话
- **标注**：加 [停顿]、[加重]、[轻松] 语气提示
- **开头**：不要自我介绍，用反常识或数据冲击直接开始
- **结尾**：一句行动号召或金句
- **删掉**：所有书面表达、复杂从句、引用出处

### 短内容 → 长文
- **字数**：扩展到 800-2000 字
- **结构**：补充论据、案例和过渡
- **语气**：从口语调整为"有深度的聊天"，可以用更长的句子
- **加小标题**：分段更清晰
- **补充**：每个要点加 1-2 个支撑案例或类比
- **结尾**：比短内容更有深度的收束

### 任意 → Twitter/X
- **字数**：≤ 280 字（单条）或 Thread 格式（每条 1 个点）
- **结构**：提取最有冲击力的 1 句判断作为首条
- **Thread 格式**：首条是 Hook → 2-5 条展开 → 末条是结论或 CTA
- **语气**：极简、有力、每个字都有信息量
- **删掉**：所有修饰词、过渡句
