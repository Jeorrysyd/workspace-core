# workspace-core

把个人笔记变成可发布内容的 AI 工作流。

> 人做决策，AI 做执行。每一步结束，AI 给判断、给选项、交还球权。

```
发现 ────────→ 角度 ────────→ 创作
扫笔记·拉信息源   钩子·立场·论据·骨架   生成·审计·精修·标题·适配
选题卡片·可行性    质疑压测·参考分析     多格式·跨平台
```

## 为什么做这个

内容创作者的日常：笔记记了一堆但不知道写什么，选题靠灵感，动笔靠硬磨，写完不确定质量。

这个工具把"从笔记到成品"拆成 3 步，每步 AI 辅助，但**决策权始终在你**——AI 不替你选题，不替你定立场，不替你决定发还是不发。

## 快速开始

```bash
git clone https://github.com/jeorrysyd/workspace-core.git
cd workspace-core
npm install
cp .env.example .env
```

编辑 `.env`，填入你的 API Key：

```env
AI_PROVIDER=anthropic-api
ANTHROPIC_API_KEY=sk-ant-你的key
NOTES_DIR=~/你的笔记文件夹    # Obsidian / Logseq / 任何 Markdown 文件夹
```

```bash
npm start
# 打开 http://localhost:3456
```

> Claude Code 用户可以设 `AI_PROVIDER=claude-cli`，不需要 API Key。

## 3 步生产线

### 1. 发现 (Discover)

从个人笔记和外部信息源中挖选题，直接输出带可行性评估的选题卡片。

- **笔记扫描** — 选时间范围，AI 从你的笔记中提取有内容潜力的方向
- **信息雷达** — 拉取 X/Twitter builders 动态，发现"外网已火、中文没人讲"的信息差
- **自由发散** — AI 分析你的所有记录，找出反复出现但从未写成内容的暗流
- **关键词追踪** — 追踪一个想法在你记录中的演变轨迹
- **手动输入** — 已经有想法？直接输入，跳过扫描

每个选题卡片包含：可行性评估、建议方向、目标读者、核心素材来源。

### 2. 角度 (Angle)

生成**角度卡片**——这是整条生产线的核心产物：

| 维度 | 内容 |
|------|------|
| 钩子 | 5 种 Hook 公式，各生成候选（反常识/数据冲击/场景代入/悬念/结果先行）|
| 立场 | 核心观点 + 三重检验（删除测试/替换测试/反对测试）|
| 论据 | 3-5 个案例，S/A/B/C/D 五级可信度标注 |
| 骨架 | 内容结构（≤7 步），要求因果递进不能平行罗列 |
| 风险 | 可能的反对意见和应对策略 |

两个加强工具：
- **质疑模式** — 5 维度压力测试（读者停留/独特性/证据强度/结构完整/风险盲点），给出 ✅/⚠️/❌ 判断
- **参考稿分析** — 粘贴一篇你喜欢的稿件，AI 提取其结构骨架供复用

### 3. 创作 (Create)

选格式，AI 基于角度卡片生成内容，内置审计和扩展工具：

**5 种输出格式：**

| 格式 | 特点 |
|------|------|
| 短视频口播稿 | 60-90 秒，口语化，钩子→案例→立场→行动 |
| 短内容图文 | 标题有钩子感，每段有金句，结尾带 tag |
| 深度文章 | 说人话、有温度、有洞察，带结构标注 |
| 学术风格 | 严谨论证，引用驱动 |
| 商业方案 | 痛点→方案→证据→行动 |

**内置质量审计（3 维度，各 10 分）：**

1. **人话指数** — 是否像真人写的（有口癖、不完美的表达、具体细节）
2. **金句密度** — 每 300 字是否有可截图传播的句子
3. **开头杀伤力** — 前 3 句能否留住读者

**扩展工具：**
- **一键精修** — AI 扫描 AI 味、优化开头、强化金句、清理结尾
- **标题生成** — 6 种公式（数字/反常识/结果/悬念/代入/对比），10 个候选 + Top 3 推荐
- **跨平台适配** — 一篇内容一键改写为其他平台格式（长文↔短内容↔短视频↔Twitter）

## 设计理念

**每步可独立，整体可连贯。** 有选题了？从角度开始。有现成稿子？直接创作步骤里审计和精修。上一步的输出自动填入下一步，但不强制。

**球权原则。** 每步 AI 完成产出后，不只是丢结果给你——它会给出判断和建议，提供 2-3 个明确的下一步选项，并标注"球在谁手里"。你永远知道下一步该做什么。

**个人化是核心。** 在笔记文件夹里放一个 `soul.md`（你的个人画像）和 `memory.md`（你的记忆），AI 会用它们理解你的风格和偏好。用得越久，AI 越懂你。

**极简技术栈。** Express + 原生 JS，无框架、无数据库、无构建步骤。项目数据存为 JSON 文件。`npm install && npm start` 就能跑。

## 配置

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `AI_PROVIDER` | `anthropic-api` 或 `claude-cli` | `claude-cli` |
| `ANTHROPIC_API_KEY` | Anthropic API Key | — |
| `NOTES_DIR` | Markdown 笔记文件夹路径 | — |
| `OWNER_NAME` | 注入到 AI 提示词中的名字 | `用户` |
| `PORT` | 服务端口 | `3456` |

## Skill 系统

9 个 AI 提示词模板驱动整条生产线：

| Skill | 用途 |
|-------|------|
| `topics.md` | 选题生成（7 种内容类型决策树）|
| `analyze.md` | 笔记分类（5 种类型识别）|
| `angle.md` | 角度卡片设计（Hook 公式 + 立场检验 + 论据分级）|
| `challenge.md` | 角度压力测试（5 维度）|
| `draft.md` | 内容生成（7 种格式模板 + 平台适配）|
| `polish.md` | 质量审计（3D）+ 精修 |
| `headline.md` | 标题生成（6 种公式 + 平台优化）|
| `adapt.md` | 跨平台适配（4 条改写路径）|
| `select.md` | 选题可行性分析（兼容旧版）|

## 项目结构

```
workspace-core/
├── index.html                    # 单页应用入口
├── css/base.css                  # 设计系统 + Pipeline 样式
├── js/
│   ├── app.js                    # 模块注册 + 事件总线
│   ├── api.js                    # HTTP + SSE 流式客户端
│   └── shared.js                 # 共享 UI 工具
├── modules/pipeline/
│   └── index.js                  # Pipeline 前端（3 步 UI + 项目管理）
└── server/
    ├── index.js                  # Express 入口
    ├── routes/pipeline.js        # 统一 API（项目 CRUD + 3 步 + 扩展工具 + 信息源）
    ├── services/
    │   ├── ai-provider.js        # AI 供应商工厂（可切换）
    │   ├── storage.js            # 文件存储（项目 + 草稿）
    │   ├── notes.js              # Markdown 笔记读取
    │   └── memory.js             # 个人记忆存储
    └── skills/                   # 9 个 AI 提示词模板
        ├── topics.md             # 选题生成
        ├── analyze.md            # 笔记分类
        ├── angle.md              # 角度设计
        ├── challenge.md          # 角度压测
        ├── draft.md              # 内容生成
        ├── polish.md             # 审计 + 精修
        ├── headline.md           # 标题生成
        ├── adapt.md              # 跨平台适配
        └── select.md             # 选题分析（兼容）
```
