### Latent Space: Dreamer: the Agent OS for Everyone — David Singleton（第三视角：Builder 经济与平台设计）

**嘉宾**: David Singleton | Co-founder & CEO @ Dreamer (prev. CTO @ Stripe, Early Android team @ Google)

> 注：Dreamer 播客在 3/22 和 3/23 的 Daily Scout 中均有深度覆盖。本期专注提炼一个前两期尚未展开的核心议题：**当 AI Agent 平台走向生态化，Builder 经济（Creator Economy for AI）如何构建？**

**核心洞察：**

**Tool Builder 变现机制：AI 生态的第一个"独立开发者"模型**

David 在播客中首次公开透露：Dreamer 平台上的 Tool Builder（工具开发者），可以按照工具被 Agent 调用的次数获得实际收益。这个设计直接类比了 App Store 的开发者分成，但针对的是 AI 时代的"工具层"而非"应用层"。目前 MCP（Anthropic 的模型上下文协议）和各种 AI 工具框架中，绝大多数工具是开源社区无偿贡献的，这在可持续性上存在隐患——优质工具的维护缺乏经济激励，长期可能导致质量下滑或开发者流失。Dreamer 的付费 Tool 模型，如果能规模化，可能催生出 AI 时代第一批真正意义上的"工具开发者"职业群体。

**"Builders in Residence" 项目：AI 生态的创意人才发现机制**

类比艺术驻留（Artist in Residence）概念，Dreamer 推出了 Builder in Residence 项目：邀请最具创意的 Agent 构建者加入，先快速开放 wait list，对于最出色的人才，直接付费让他们构建 Agent，并与核心团队深度合作。David 的原话："就像乐高——我们搭好了基础积木，如果你需要一种我们还没有的形状，我们来为你定制。" 这是平台发展早期加速生态内容建设的一种巧妙机制：不是自上而下定义所有用例，而是用激励和支持发现社区中的创意边界。

**Sidekick 作为"权限内核"：Agent 之间信任架构的工程实现**

从技术深度看，David 解释了 Dreamer 中多 Agent 协作的核心安全机制：各 Agent 之间不直接通信，必须通过 Sidekick（核心个人 Agent）中转。Sidekick 掌握所有权限边界的"宪法"——它知道用户授权了哪些工具、哪些数据访问范围，任何 Agent 的请求都会经过 Sidekick 的合规校验。这在架构上解决了 "多 Agent 协作 + 隐私安全" 之间的根本张力：让 Agent 能力强大的同时，通过信任架构确保边界不被穿越。对于正在构建 Multi-Agent 系统的团队，这是一个值得借鉴的架构模式。

**LLM.txt 文件：AI 时代的新型开放数据标准**

David 在演示中提到，他之所以能为 Swyx 的 AI Engineer 大会快速构建 Agent App，是因为 Swyx "前瞻性地把大会数据以 LLM 可读的方式放到了网页上——有 LLM.txt 文件，有 JSON 格式的 Session feed"。这是一个重要的生态标准信号：LLM.txt 正在成为网站向 AI Agent 公开数据的标准格式，类比 robots.txt 告诉爬虫哪些内容可访问，LLM.txt 告诉 AI Agent 如何理解和使用网站内容。这个标准的推广意味着：具有 Agent 友好数据结构的网站，将能被 AI Agent 更有效利用，在 Agent 生态中获得更高的曝光和使用频率。

**为什么重要：** 本期分析的这些 Dreamer 细节（Builder 变现 + Builders in Residence + Sidekick 权限架构 + LLM.txt 标准），共同描绘了 AI Agent 平台进入"可持续生态"阶段需要解决的核心问题。对于正在思考如何构建 AI 工具生态、如何做 MCP 服务货币化、如何设计多 Agent 信任架构的团队，这些是高浓度的实战参考。
